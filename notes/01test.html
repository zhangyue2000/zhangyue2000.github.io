Chapter 1 A Primer in Column Generation

  • 引言

    • 如果受限问题的解对原问题可行但不是最优：假设可以设计一个算法A，其可以找到一个变量，而此变量因出现在原问题的最优解中
    • 之后将此变量加入到受限问题，重新求解
    • 最后，基于新的解运行算法A来得到另一个新的变量，直到A发现没有新的变量需要加入，也就是说现在的解对原问题就是最优的

Column generation

  • 基础

    ZMP∗=min ∑j∈Jcjxjsubject to ∑j∈Jajxj=bxj≥0,j∈J

  • 定义A=[aij]m×n，x=[xi]n，b=[bi]m

  • 基变量记为xBV，非基变量记为xNBV

  • 变量系数记为cBV，非基变量记为cNBV

  • B为m×m的矩阵，第j列是BVj对应的列

  • aj是约束中变量xj对应的列

  • N是m×(n−m)的矩阵，其列是非基变量的列

 

  • 问题可写作
z=cBVxBV+cNBVxNBVs.t. BxBV+NxNBV=bxBV,xNBV≥0
  • 用B−1乘上约束BxBV+NxNBV=b得到xBV+B−1NxNBV=B−1b

 

  • 对于初始LP确定最优表第0行

    • 首先用向量cBVB−1乘上约束BxBV+NxNBV=b得到cBVxBV+cBVB−1NxNBV=cBVB−1b
    • 注意到目标函数为z−cBVxBV−cNBVxNBV=0
    • 因此,z+(cBVB−1N−cNBV)xNBV=cBVB−1b

  • 令c¯j为变量xj在最优表第0行的系数

    • c¯j=cBVB−1aj−cj

  • c¯j被称为判别数(reduced cost)，其含义为对于每个非基变量，如果其从0开始增加，目标Z的改善率

    • 对于最小化问题，如果存在一个非基变量的判别数为正，说明当前解仍非最优

      z+(cBVB−1N−cNBV)xNBV=cBVB−1b

 

 

  • 而对于大规模问题而言，通常大部分数列并不需要进基，因此不必产生这些无用的列

  • 而找到具有正判别数的变量xj的方法为Subproblem

    minimize −c¯jwhere the minimization is over all j(对于每一列都应该满足的约束)

  • 不再去计算每一个c¯j,直接求解上述问题得到

  • 若最小的判别数仍小于等于0,则得到最优解;否则将该最小的判别数对应的变量进基

  • 而另一个有趣的性质是

  • −c¯j=cj−cBVB−1aj中cBVB−1是RMP对偶问题的对偶变量

  • 因此,我们也可以直接求解RMP问题的对偶变量,然后找到需要的判别数和它对应的变量,变量的系数加入到RMP中去继续求解

 

  • 如果需要出基
  • 计算B−1aj得到当前表中需要进基的变量的系数
  • 计算B−1b得到当前表中的右端项
  • 根据上述两项的比值确定要出基的变量在哪行
  • 新的B1−1=E0B0−1, E0=[1,00,10,0]+[B−1

    • G=(V,A),V,A=nonempty sets of nodes and arcs
    • A path P=(e1,...,ep) is a finite sequence of arcs ( some arcs may occur more than once ) where the head node of ei∈A is identical to the tail node of ei+1∈A for all i=1,...,p−1.
    • a path can be written as P=(v0,v1,...,vp), whose length is p

 

  • resource constraint

    • (minimal) resource comsumptions and resource intervals (e.g. the travel time and time window)
    • R：the number of resources;
    • T=(T1,...,TR)T∈RR：a resource vector, and its components resource variables
    • T is said to be not greater than S=(S1,...,SR)T∈RR if the inequality Ti≤Si holds for all components i=1,2,...,R
    • resource intervals ( also called resource windows ) associated with a node i∈V are denoted by [ai,bi] with ai,bi∈RR,ai≤bi

 

 

Chapter 4 Branch and Price Heuristics: a Case Study on the Vehicle Routing Problem With Time Window

 

  • In case the master problem contains integrality constraints on some of its variables, column generation and branch-and-bound are combined: This is called branch-and-price
  • Branch-and-price provides the user both with a lower bound and integer solutions
  • Branch-and-price is known for providing tight lower bounds but it has sometimes difficulties to generate rapidly good solutions because the linear relaxation of the master problem rarely has an integer solution

 

 

  • Application to the vehicle routing problem with time window

    • Branch-and-price model and solution techniques

    • Let {1,...,n} be the set of customers

    • For each feasible route r, let xr be the variable defined by

      xr={1 if route r is used in the solution0 otherwise 

    • The VRPTW is then written as

      min∑r∈Rcrxrs.t. ∑r∈Rδirxr=1, ∀i=1,...,nxr∈{0,1}, ∀r∈R

      where R is the set of all feasible routes with respect to the capacity and time windows contraints, and δir=1 if customer i is visited by route r, and 0 otherwise

  • Decomposition into master problem and subproblem

    • The first difficulty of this model is that the number of feasible routes grows exponentially with the number of customers

    • Master problem

      • min∑r∈R^crxr∑r∈R^δirxr=1, ∀i=1...,nxr∈{0,1}, r∈R^

        where the R^ is the set of already generated columns

      • we solve in fact the continuous relaxation of the master problem, replacing xr∈{0,1} with the constraints 0≤xr≤1,∀r∈R^

    • Subproblem

      • minminr∈R⁡ cr−∑i=1nπiδir

        where πi is the dual price.

  • Branch-and-price and branching strategy

    • The second difficulty of this model is that, xr variables must take integer values in feasible solutions. Therefore, the problem is solved with branch-and-price
    1. Start with an initial pool of columns, for example generated by a simple heuristic
    2. Solve the continuous relaxation of the master problem
    3. Solve the subproblem with the dual values updated at step 2, and attempt to generate several constrained shortest paths with negative reduced costs
    4. Iterate steps 2 and 3 until no more new routes with negative reduced cost can be generated
    5. If the solution of the continuous relaxation of the master problem is not integer, branch and iterate steps2 and step 3 at each node
    • let x∗ be the optimal solution of the relaxed master problem after the last subproblem iteration at the current node.
    • If x∗ is integer, no branching is necessary
    • If x∗ is not integer, let t0={i0=depot,i1,i2,...,ip+1=depot} be the route such that xt0 is the variable with the most fractional value in x∗
    • xt0<1，hence for each k∈{1,...,p}, there exists other routes that cover ik and take a non-zero value in x∗
